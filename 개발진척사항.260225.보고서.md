# 🚀 자체 개발 진척 사항 보고서 (2026-02-25)

## 📌 주요 작업 요약

### 1. 데스크톱 앱 (Tauri) 전환 및 미디어 편의성 개선
* **Tauri 앱 기반 설정**: 웹 브라우저 기반으로 구동되던 게임을 완전한 독립형 Windows 데스크톱 애플리케이션으로 패키징하기 위한 Tauri 초기 환경 구축 완료. 브라우저 창 없이 쾌적한 플레이가 가능해졌습니다.
* **오디오 자동 재생 (Autoplay) 이슈 해제**: 브라우저 보안 정책상 막혀있던 "사용자 클릭 전 오디오 재생 불가" 제약을 Tauri 웹뷰 설정(`additionalBrowserArgs: --autoplay-policy=no-user-gesture-required`)을 통해 강제로 풀업하여, 앱 실행 시 인트로부터 곧바로 웅장한 음악이 재생되도록 권한 구조를 변경 완료했습니다.

### 2. IndexedDB 세이브 데이터 로드 오류 완전 정복
* **치명적 크래시 및 렌더링 에러 해결**: 이전에 게임을 저장 및 불러올 때 `JSON.stringify()` 과정에서 아이템의 프로토타입(Prototype)과 동적 속성인 `name`, `desc` 등이 삭제되어 상점 및 보관함 화면에서 게임이 터지던 (TypeError) 현상을 근본적으로 차단했습니다.
* **하위 호환성 복구 알고리즘 도입**: 기존에 이미 깨진 상태로 저장되었던 `id` 없는 아이템 세이브 파일들도 지우지 않고, `instanceId` 문자열을 역추적(split/pop)하여 원본 아이템의 `id`를 성공적으로 유추합니다. 이후 원본 아이템 껍데기를 다시 씌워 정상적인 번역 데이터와 효과를 모두 사용할 수 있도록 하위 호환성을 확보했습니다.

### 3. 시간 흐름 안정화 및 이벤트 발생 (Event System) 엔진 강화
* **휴식 중 시간 흐름 동기화**: Safe House에서 'REST(휴식)'를 눌렀을 때 체력과 날짜 타이머만 겉돌고 이벤트 검사(processEvents)가 한 번도 호출되지 않아 각종 임대료 계산, 소득세 계산 등 월 스케줄이 끊기던 문제를 완벽히 동기화(`App.vue` -> `staminaSystem.js` -> `timeSystem.js` 사이클 완성)시켰습니다.
* **시작일 엇갈림 패치**: `store.js`의 `gameTime` 시작일은 1월 20일로 변경되었으나 `eventSystem.js`는 아직 10월 20일에 멈춰있어서 음수 값이 나오던 버그를 색출해 `2057-01-20` 값으로 통일했습니다. 

### 4. 고도화된 스크립트 이벤트 구조 기틀 형성
* **일회성 및 조건형 연출 (`repeatable: false`)**: 이전에는 렌트비/소득세처럼 정해진 시간에 "무한 반복" 되는 이벤트만 있었지만, 이제는 `condition()` 함수 검사 개념과 `repeatable` 판별 기법을 도입하여 조건이 달성될 때 **딱 1회만** 작동하는 연출성 이벤트를 완벽 지원합니다.
  * **[신규 추가 구현] 거친 친구들의 조언 이벤트 (`you_get_some_sleep`, `big_lose`)**: 체력이 너무 낮거나(`store.stamina <= 25`), 파산 직전 소지금(`store.bankroll <= 5000`)에 다다를 경우, NPC가 문자와 함께 1회용 긴급 지원금/회복을 보내주어 게임 오버를 면하게 해주는 극적인 시스템을 적용했습니다!
* **결제 수령 (`RECEIVE`) 시스템**: 플레이어가 단순히 돈을 지출(Pay)하는 구조에서, 누군가로부터 위로금이나 상금을 받아 내 뱅크롤을 증가시킬 수 있는 긍정적인 메시지 수령 기능도 무사히 추가 및 버그 검수(`amount` 스펠링 수정)를 마쳤습니다.

### 5. 인게임 튜토리얼 멘토(Mentor) 시스템 및 동적 대사 구현
* **맥스(Max) 멘토링 기능 추가**: 튜토리얼 성격의 맵(`free_street_shop_with_max`, `micro_warehouse_with_max`)에서 맥스가 플레이어의 액션(폴드, 콜, 체크 등)과 승패를 관찰하고 텍사스식 포커 팁을 던져주는 전용 멘토 시스템(`Max(Mentor)`)을 스폰 로직과 함께 도입했습니다. 
* **배팅 사이즈 기반 동적 훈수**: 플레이어가 레이즈할 때, 팟(Pot) 비율을 수학적으로 계산해 '미니 레이즈(35% 미만)', '스탠다드 레이즈', '오버 벳(120% 이상)', '올인(All-in)' 등 구체적인 배팅 강도에 맞춰 각기 다른 조언을 하도록 `gameEngine.js`의 추적 로직을 고도화했습니다.
* **쇼다운(Showdown) 결과 정밀 분석 로직 개선**: 다자간(Multi-way) 승부 시 사이드 팟(Side Pot) 결과가 무승부(Chop)로 오작동하던 기존 로직을 갈아엎었습니다. 이제는 **플레이어가 이번 판에 실제로 판돈으로 걸었던 칩(totalWagered) 대비 순수익(Net Profit)**을 계산하여 잃은 돈이 큰지, 딴 돈이 큰지를 판별하고 그에 맞는 멘토 피드백을 출력합니다.

### 6. AI 챗 팝업 UX 개선 및 코어 버그 수정
* **대사 길이에 따른 말풍선 지속 시간 자동 조절**: 멘토의 조언 등 긴 텍스트가 화면에 노출될 때, 글자 수에 비례하여 말풍선이 유지되는 시간(Duration)을 동적으로 연장시켜주는 수식을 `aiEngine.js`에 적용해 유저의 가독성을 크게 개선했습니다.
* **초기화 순서 충돌 및 스폰 버그 완벽 제어**: `GameEngine`의 생성자(Constructor)에서 애써 세팅한 튜토리얼 변수(`this.isTutorial`, `this.mentor`)가 나중에 다시 `false`와 `null`로 덮어씌워지던 타이밍 버그를 잡았으며, `zone.js`의 `partners` 배열 매핑도 `CLASSES_PARTNER` 데이터와 정확히 물리도록 연동시켰습니다.
