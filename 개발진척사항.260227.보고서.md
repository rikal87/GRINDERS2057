# 개발 진척 사항 보고서 (2026.02.26 - 2026.02.27)

## 📌 주요 업데이트 요약

지난 이틀간 게임 내 **파트너 시스템(Partner System)의 핵심인 '계약(Contract)' 시스템**을 중점적으로 개발 및 고도화하였으며, 관련 UI/UX 개선 및 치명적 버그 수정이 이루어졌습니다.

### 1. 🤝 파트너 계약 시스템 (Contract System) 전면 개편
단순 호감도 기반의 조력자를 넘어, **리스크와 리턴을 주고받는 비즈니스 파트너십** 모델을 게임 내 로직으로 완전 구현했습니다.

*   **계약 종류 세분화:**
    *   `GAMBLING_WITH_ME`: 카지노 동행 (테이블에 파트너 스폰)
    *   `SHARE_BENEFIT`: 수익률 스왑 (비율에 따른 세션 수익/손실 분배)
    *   `BANKRUPT_RESCUE`: 파산 보험 (파산 시 초기 뱅크롤 지원)
    *   `A_DATE_WITH_YOU`: 데이트 (지속적인 호감도 변동)
*   **패널티 및 쿨다운 메커니즘 도입:**
    *   계약 파기 시, 게임 내 시간으로 **72시간(3일) 동안 재계약 불가** 쿨다운 적용 (`breakContract`).
    *   계약 체결 직후 발생하는 재수 없는 연패 상황을 방지하기 위한 **24시간 정산 유예기간(Grace Period)** 적용 (`signContract`).
*   **하이드레이션(Hydration) 로직 구축:**
    *   과거 세이브 데이터(JSON 오브젝트)에서 `Contract` 클래스 인스턴스로 안전하게 복원하는 로직 작성.
    *   `registerPartner`를 통해 로딩 시 새로운 계약 유형(예: 나중에 추가된 데이트 계약 등)을 기존 세이브 파일 파트너에게도 버그 없이 병합(Merge) 되도록 호환성 패치 완료.

### 2. ⚖️ 동적 수익 분배 및 호감도 스케일링 (SHARE_BENEFIT 고도화)
수익 분배 계약 정산 시, **비율(Ratio)과 파트너 뱅크롤 상황에 따른 동적 호감도 증감 공식**을 설계 및 반영했습니다.

*   **세션 잔고 추적 버그 픽스:** 
    *   매 턴(틱)마다 파트너의 전 재산이 중복 누적되어 엄청난 채무가 발생하던 치명적 버그 수정.
    *   이제 순수 **세션 변동금(`netWorthChange`)** 단위로만 정확히 누적 정산 (`sessionNetWorth`).
*   **합리적인 관계도 증감 로직(Ratio Modifier) 설계:**
    *   `impactPercentage`: 플레이어가 나눠준 수익이 파트너의 '전 재산 대비' 얼마나 큰 체감도를 가지는지 계산.
    *   `ratioModifier`: 플레이어가 수익을 독식하는 악덕 계약(0.8 등)일 경우 호감도를 깎고, 양보하는 혜자 계약(0.2 등)일 경우 호감도를 대폭 올려주는 동적 방정식 `1.0 + ((0.5 - ratio) * 4)` 적용.

### 3. 🎭 페르소나 맞춤형 이벤트 스크립트 작성
파트너(Max, Florence)와 계약 체결 및 파기 시 발생하는 서사(Dialog)를 각 캐릭터의 성격과 배경 설정에 맞춰 100% 반영했습니다.

*   **맥스 (Max) - 텍사스 출신 다혈질 베테랑:**
    *   "텍사스 콤비 결성!", "사나이 최고의 보험" 등 마초적인 말투와 의리 지향적인 대사 패턴 적용.
*   **플로렌스 (Florence) - 라스베가스 출신 차가운 계산가 (TAG):**
    *   "위험 분산 포트폴리오", "최후의 안전장치" 등 철저하게 EV(기댓값)와 헷징(Hedging)을 운운하는 금융/비즈니스적 대사 적용.
    *   데이트 계약에서 미세하게 감정을 내비치는 츤데레 텍스트 추가.

### 4. ⚙️ 기타 버그 픽스 및 UI 개선
*   **아이템 효과 합산 에러 ( items.js ):**
    *   읽기 전용 Getter 함수(`calcValue` 등)를 가진 스킬의 수치를 상점 진열 시 강제 합산하려다 뜨는 `TypeError`를 해결하기 위해 `materializeItem` 내부 병합 과정에 `try...catch` 방어 코드 삽입.
*   **SafeHouse.vue UI 연동:**
    *   계약 쿨다운(`contract.cooldown > 0`) 상태일 때 CSS 클래스 바인딩 에러 픽스 (`:class="{ active: ... }"`) 및 비활성화 상태 표현 기능 추가 (SIGN / CANCEL 버튼 제어 연동 완료).
*   **GameEngine.js 카지노 스폰 연동:**
    *   `GAMBLING_WITH_ME` 계약이 체결(`active === true`)된 파트너만 테이블에 스폰되도록 연동 완료.
