# 개발 진척 사항 보고서 (2026.03.01)

## 📌 주요 업데이트 요약

오늘은 `SafeHouse` UI의 **모듈화 및 컴포넌트 분리**를 통해 코드 복잡도를 크게 낮추었으며, 게임의 핵심 콘텐츠인 **파트너 채무(Debt) 및 계약(Contract) 시스템**을 전면 개편하여 안정성과 몰입감을 극대화했습니다. 또한, 신규 아이템 추가 및 기존 아이템 밸런싱 작업과 더불어, **파트너 맥스(Max)의 전용 스토리 이벤트와 페르소나 대사**를 한층 더 심도 있게 구현했습니다.

### 1. 🏗️ SafeHouse UI 컴포넌트 모듈화 및 채무 UI 개편
너무 방대해진 `SafeHouse.vue` 파일의 유지보수성을 높이기 위해, 성격이 독립적인 모달(Modal) 창들을 개별 컴포넌트로 완벽히 분리했습니다.
* **컴포넌트 분리:**
  * `AiAgentPopup` (AI 에이전트 서브루틴 세팅)
  * `SkillSelectorModal` (스킬 장착 팝업)
  * `PlayStatsModal` (플레이어 통계표)
  * 위 모달들을 분리하여 `App.vue`에서 최상위로 띄울 수 있도록 구조를 개선했습니다.
* **SafeHouse 내부 파트너 UI 개선:**
  * `CURRENT_DEBT` (현재 채무 상태)를 명시적으로 확인할 수 있도록 패널을 추가.
  * 빚을 갚을 때 숫자를 직접 입력하는 방식에서, **Ratio(슬라이더)를 통제하여 퍼센티지 단위로 안전하게 상환**(`sendDebtRepayment`)할 수 있도록 UI/UX를 개선.
  * 읽지 않은 메시지 뱃지 등 알림 인터페이스 최적화 진행.

### 2. 🤝 파트너 채무(Debt) 상환 로직 및 Contract 데이터 안전성 확보
파트너 간의 금전적 상호작용(빚을 지거나 갚는 행위)과 계약(Contract)이 브라우저 로컬 스토리지에 저장될 때 발생하던 **직렬화(Serialization)/반응성(Reactivity) 유실 문제**를 뿌리 뽑고 로직을 고도화했습니다.
* **채무 상환 로직(`debtRepayment`) 개선:**
  * 플레이어가 파트너의 빚을 갚아주거나(`toPlayer = false`), 파트너가 플레이어에게 빚을 갚을 때(`toPlayer = true`) 부호를 동적으로 처리하도록 **하나의 통합 계산식으로 단일화 (분기점 제거)**.
  * 파트너가 플레이어에게 빚을 갚을 때(`actualRepaid > 0`), **상환한 금액에 비례하여 파트너의 관계도(Relationship)가 상승**하는 시스템 추가. (은혜를 갚으며 사이가 돈독해짐)
* **Factory Function (팩토리 함수) 도입 (`store.js` 연동 개선):**
  * 기존 `class Contract` 기반 인스턴스는 로컬 스토리지를 거치면서 Proxy 추적이 끊기고 속성이 날아가는 치명적 문제가 있었습니다.
  * 이를 `ContractShareBenefit()`, `ContractBankruptRescue()` 등 **순수 객체를 반환하는 팩토리 함수 패턴**으로 전면 교체하여, Vue UI 렌더링(반응성)과 세이브 데이터의 무결성을 동시에 확보했습니다.
  * 기존 세이브 파일에서도 에러가 터지지 않게 빈 속성을 채워주는 `createContractObject` 헬퍼 함수 기반의 `hydratePartners` 리팩토링 완료.

### 3. 💬 맥스(Max) 페르소나 대사 강화 및 미션(Story) 이벤트망 확장
메인 스토리의 첫 번째 징검다리이자 텍사스 마초 친구인 **맥스**의 성격과 스토리를 더욱 생생하게 연출했습니다.
* **신규 빚/파산 관련 메시지 구현:**
  * `BANKRUPT_RESCUE_REPAYMENT_DONE`, `DEBT_REPAYMENT_DONE` 등 **구조 자금을 갚았거나 빚을 모두 청산했을 때** 맥스가 보내는 유쾌하고 의리 넘치는 전용 대사들을 `ko`/`en` 완벽 지원.
  * 상환 시점에 관계도가 나쁘면(`_LOW_RELATIONSHIP`), 다소 쌀쌀맞게 비즈니스적으로 돈만 돌려주고 끊는 디테일한 분기점 추가.
* **메시지 타입 신설 및 메인 스토리 분기:**
  * 메인 퀘스트 전용 알림을 시각적으로 구분하는 `MESSAGE_TYPE.MISSION` 상수 신설.
  * `MAIN_STORY_1_2_MEET_AT_CLUB` 이벤트 진행 중, 클럽에서 일정 금액 이상 벌었는지(Success) 혹은 잃었는지(Failed)에 따라 분기를 가르는 트리거(`gameEngineEventAdaptor.js`) 적용.

### 4. 🎒 아이템 추가 및 기존 밸런싱 (Tier & Rarity 조정)
게임 플레이에 변수를 주는 상점/인벤토리 아이템을 일부 재배치하고 신규 아이템을 고안했습니다.
* **신규 아이템:** 
  * `카지노 프로모션 정보 수집기 (T1)`: 싸게 접근할 수 있도록 Blind Discount 효과 제공.
* **밸런스(Tier/효과) 조정 내역:**
  * `크라이오 민트 디스펜서`: 기존 시간 지연(T1) -> 수면 부족을 깨우는 **강력한 스태미나 회복제(T5, 390,000 CR)**로 스펙 격상.
  * `LT 회복(lt_recovery)` 희귀도 Uncommon 변경.
  * `LT 회복량 증가` 및 `블라인드 할인`은 초반 체감 향상을 위해 Common 등급으로 하향.
  * `더블 다운` Uncommon으로 재조정.
