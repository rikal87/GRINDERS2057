# 개발 진척 사항 보고서 (2026.03.16)

## 📌 주요 업데이트 요약

이번 업데이트에서는 **게임 엔진의 안정성 확보**, **아이템 효과 시스템의 전면적 보수**, 그리고 **AI 전략 로직의 정교화**를 중점적으로 진행했습니다. 특히 세션 종료 시 발생하던 각종 데이터 오차(NaN)와 크래시 문제를 해결하여 완성도를 높였습니다.

1. **게임 엔진 안정성 및 예외 처리 강화**: 세션 전환 및 종료 시 발생하던 `TypeError`를 해결하고, 초기 칩 계산 시의 `NaN` 오류를 완벽히 제거했습니다.
2. **아이템 시스템 확장 및 효과 로직 개편**: 신규 아이템 '주쉭(Stonk)'과 '호구(Sucker)'를 추가하고, 기존 효과들의 발동 조건과 계산식을 더욱 전략적으로 개선했습니다.
3. **AI 블러프 전략 및 핸드 공개 로직 최적화**: 보드 텍스처와 스트리트에 따른 AI의 블러프 빈도를 조정하고, 특정 아이템 효과에 따른 핸드 강제 공개 시스템을 구현했습니다.
4. **통계 및 리포트 시스템 수정**: 세션 종료 후 추가 수익(아이템 보너스)이 리포트에 정상적으로 표시되지 않던 반응성 문제를 해결했습니다.

---

### 1. ⚙️ 게임 엔진 안정성 및 데이터 정밀도 (`gameEngine.js`, `playRecordStats.js`)
엔진 내부의 비동기 레이스 컨디셔너와 데이터 초기화 순서를 재설계했습니다.
* **초기 칩 계산 오류(NaN) 수정**: 플레이어 생성 시 `buyInMultiply` 계산 순서를 앞당겨, 아이템 보너스가 포함된 초기 칩이 `NaN`으로 설정되는 문제를 해결했습니다.
* **통계 기록 시스템 예외 처리**: 세션 통계(`sessionStats`)가 팝업 종료 후 null이 된 상태에서 통계 기록 함수가 호출되어 발생하던 `TypeError`를 방지하기 위해 전역적인 Null Check 로직을 도입했습니다.
* **안정적인 게임 종료 제어**: `exited` 플래그를 도입하여 세션 종료 후 비동기적으로 새로운 핸드가 시작되지 않도록 차단함으로써 데이터 오염을 방지했습니다.

### 2. 💎 아이템 효과 및 경제 시스템 고도화 (`gameEngineEventAdaptor.js`, `items.js`)
기존의 단순 확률형 보너스에서 실제 플레이 성과와 연동되는 방식으로 로직을 고도화했습니다.
* **신규 아이템 '주쉭(Stonk)' 도입**: 
  * 기존 '주식' 아이템을 '주쉭'으로 개편하며 단순 승률이 아닌 **쇼다운 승률(W$SD/WTSD)**에 비례해 수익률이 결정되도록 물리 법칙을 적용했습니다.
* **'블러프를 보여줘(Sucker)' 효과 구현**: 
  * 폴드 시점이 아닌 `round_end` 시점에 발동하도록 변경하여, 승자가 팟을 가져갈 때 일정 확률로 핸드를 강제로 공개하도록 시스템을 수정했습니다.
* **리포트 UI 반응성 수정**: 리포트 팝업에서 `ITEM_EFFECT` 항목이 0으로만 표시되던 문제를 해결하기 위해, 초기 통계 객체 생성 시 해당 필드를 명시적으로 초기화하여 Vue의 반응성 시스템에 등록했습니다.

### 3. 🧠 AI 전략 및 전투 UI 개선 (`aiEngine.js`, `PokerTable.vue`)
* **상황 인식 블러프 로직**:
  * 보드 텍스처가 'Dry'하거나 'Wet'한 상황, 특히 리버 스트리트에서의 비용 대비 효율을 고려하여 AI의 블러프 빈도(`bluffFreq`) 계산식을 정교화했습니다.
* **핸드 공개 시스템(Hole Cards Reveal)**:
  * 특정 아이템(호구 등)이나 쇼다운 상황에서 상대방의 핸드를 시각적으로 노출할 수 있도록 `showHoleCards` 속성을 추가하고 `PokerTable` 컴포넌트와 연동했습니다.

### 4. 📊 성능 리포트(Performance Report) 시각화 개선
* **아이템 수익 가독성**: 세션 리포트의 경제 지표 섹션에서 아이템을 통해 얻은 추가 수익이 실제 뱅크롤 증가분과 일치하게 표시되도록 동기화했습니다.
* **메시지 로직 최적화**: 세션 결과에 따른 유저 피드백 메시지 코드가 리포트 팝업 인스턴스에 올바르게 전달되도록 개선했습니다.
