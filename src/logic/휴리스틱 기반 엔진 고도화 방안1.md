현재 휴리스틱기반 aiEngine.js는 다음과 같은 문제가있다.

1. MARGINAL(EQ:.5) 팟오즈상 콜만 해도 되는 핸드는 절대 포기하지않음
2. 레이즈 쓰레쉬홀드 개념은 조절하기 쉽지만 이 수치에 따라 일방적으로 배팅만 반복하거나 콜만 반복하게됨.
3. 따라서 이제부터 핸드 에퀴티를 기반으로 배팅을 하지만, 확률개념을 도입

페르소나 스탯이 AF 2.5, WTSD .25일 경우,
vPIP, WTSD은 배팅에 콜할 확률에 가중치를 부여한다. vPIP는 플랍상황에서 WTSD포스트 플랍에기여한다.

예시 나의 핸드가 플랍에 MARGINAL(EQ:.5)일 경우,
포스트플랍 기준 플랍에서 리버까지 배팅을 평균 1.5번을 한다, 상대의 리레이즈에도 .5확률로 콜을한다.
예시 나의 핸드가 플랍에 MARGINAL(EQ:.6)일 경우,
포스트플랍 기준 플랍에서 리버까지 배팅을 평균 2번을 한다, 상대의 리레이즈에도 .6확률로 콜을한다.
예시 나의 핸드가 플랍에 MARGINAL(EQ:.7)일 경우,
포스트플랍 기준 플랍에서 리버까지 배팅을 3번을 한다, 상대의 리레이즈에도 .7확률로 콜을한다.

상대의 리레이즈에 대한 대응 로직 예시
let action = fold
const eq = 0.7
const rnd = Math.random()
if(rnd <= eq*2*WTSD){
  if(rnd <= eq+(AF-2)*.1) action = raise
  else action = call
}else {
  action = fold || check
}

여기서 AF의 개념을 추가 하여 폴드하지 않고 콜할 확률에 AF는 콜대신 오히려 상대 리레이즈에 폴드/콜대신 리레이즈를 할 확률로 계산한다.(블러프던 밸류레이즈건)
