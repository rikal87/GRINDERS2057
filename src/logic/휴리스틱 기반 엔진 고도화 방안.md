현재 휴리스틱기반 aiEngine.js는 다음과 같은 문제가있다.

1. MARGINAL(EQ:.5) 팟오즈상 콜만 해도 되는 핸드는 절대 포기하지않음
2. 레이즈 쓰레쉬홀드 개념은 조절하기 쉽지만 이 수치에 따라 일방적으로 배팅만 반복하거나 콜만 반복하게됨.
3. 따라서 이제부터 핸드 에퀴티를 기반으로 배팅을 하지만, 확률개념을 도입

페르소나 스탯이 AF 2.5, WTSD .25일 경우,
vPIP, WTSD은 배팅에 콜할 확률에 가중치를 부여한다. vPIP는 플랍상황에서 WTSD포스트 플랍에기여한다.



프리플랍 상황에서 이제부터 자신이 받은 홀카드 에퀴티 + vPIP로 팟의 참여 여부를 결정한다(콜)

CALL이 결정된 상황일때, AF는 오히려 raise액션으로 변환될 확률로 계산한다.
다만 프리플랍의 경우 싱글레이즈팟, 3BET팟, 4BET팟에 따라 핸드레인지가 대폭 줄기 때문에
Single raise pot : eq+vPIP-.5 > 0.3 = call
3bet 팟 : eq+vPIP-.5 > 0.5 = call
4bet 팟 : eq+vPIP-.5 > 0.65 = call
같은 엄격한 커트라인이 필요하다.(수치는 조정필요)

vPIP = 0.5, AF = 3, 받은 핸드의 eq=0.5일때
const rnd = Math.random()
let action = call;
if(rnd < eq+vPIP-.5 ){
  if(rnd < (AF-2)*.1+eq) action = raise
}else {
  action = fold || check
}
의견: 위 로직 적용시 무한 레이즈를 어떤식으로 막을건지는 추가 시스템이 필요해보임

포스트 플랍의 경우

예시 나의 핸드가 플랍에 MARGINAL(EQ:.5)일 경우,
포스트플랍 기준 플랍에서 리버까지 배팅을 평균 1.5번을 한다, 상대의 리레이즈에도 .5확률로 콜을한다.
예시 나의 핸드가 플랍에 MARGINAL(EQ:.6)일 경우,
포스트플랍 기준 플랍에서 리버까지 배팅을 평균 2번을 한다, 상대의 리레이즈에도 .6확률로 콜을한다.
예시 나의 핸드가 플랍에 MARGINAL(EQ:.7)일 경우,
포스트플랍 기준 플랍에서 리버까지 배팅을 3번을 한다, 상대의 리레이즈에도 .7확률로 콜을한다.

상대의 리레이즈에 대한 대응 로직 예시
let action = fold
const eq = 0.7
const rnd = Math.random()
if(rnd <= eq*2*WTSD){
  if(rnd <= eq+(AF-2)*.1) action = raise
  else action = call
}else {
  action = fold || check
}

여기서 AF의 개념을 추가 하여 폴드하지 않고 콜할 확률에 AF는 콜대신 오히려 상대 리레이즈에 폴드/콜대신 리레이즈를 할 확률로 계산한다.(블러프던 밸류레이즈건)

### 팟오즈(Pot Odds) 통합 방안
기존에 구현된 팟오즈 개념을 확률형 모델에 다음과 같이 결합한다.

1. **승률 기반 최소 요구치(Required Equity)**:
   - `Pot Odds = callAmt / (Total Pot + callAmt)`
   - 예: 팟이 100이고 콜할 금액이 50이면 팟오즈는 `50 / 200 = 0.25 (25%)`.
   - 나의 `estimatedEquity`가 팟오즈보다 높다면 수학적으로는 '콜'이 정답이다.

2. **확률 보정 (Correction Factor)**:
   - 페르소나의 성향(`WTSD`, `vPIP`)에 따른 참여 확률(`continueProb`)을 계산한 뒤, 팟오즈가 매우 유리할 경우 이를 보정한다.
   - `finalContinueProb = Math.max(continueProb, (estimatedEquity / Pot Odds) * 0.5)`
   - 이를 통해 수학적으로 매우 유리한 상황(팟오즈가 좋은 경우)에서는 페르소나의 성향이 소심하더라도 콜할 확률을 높여준다.

3. **상대 배팅 크기에 따른 압박**:
   - 상대의 배팅이 커서 팟오즈가 나쁠 경우(요구 승률이 높음), `continueProb`를 감쇄시킨다.
   - `if (estimatedEquity < Pot Odds) continueProb *= 0.8` (폴드 유도)

의견: 팟오즈는 '수학적 정답'이고 페르소나 스탯은 '인간적 실수/성향'입니다. 이 두 수치의 비중을 실시간으로 조절하여(예: 전문가 수준 AI일수록 수학적 정답에 가중치) 밸런스를 맞추는 것이 핵심입니다.
