현재 대부분 로직이 한곳에 쌓여잇어서 상당히 지저분해지고 유지보수가 힘들어지고 있음 때문에 현재의 aiEngineAdvanced.js 참고하여

아래와 같이 새로운 aiBrainEngine.js를 만들어서 다음과 같이 세분화하길 원함


# 1. 허브 레이어(aiBrainHub.js)
아래의 각종 의사결정에 필요한 정보를 수집하여 하위 레이어에 넘겨주고 최종액션을 결정하는 허브
1-1. 현재 핸드 정보
1-2. 현재 보드 정보
1-3. 팟에 참여한 플레이어 정보
1-4. spr 정보
1-5. 빌런 정보
1-6. 수집된 정보로 최종 의사결정

# 2. 프리플랍 전략 레이어 (preflop.js)
2-1. GTORange.js에서 가져온 스타팅 핸드레인지 정보를 바탕으로 오픈레이즈, 콜, 3뱃, 4뱃+ 체크 등의 액션을 결정
2-2 아래의 빌런 프로파일링 도구를 이용하여 추가 정보 획득

# 3. 포스트 플랍 전략 레이어 (postflop.js)
3-2. 보드에 깔린 정보로 자신의 에퀴티를 추출
3-3. 팟에 참여한 각 프로파일링 정보를 조합하여 최적의 결정
3-4. 아래의 빌런 프로파일링 도구를 이용하여 추가 정보 획득
3-5. 아래의 보드 분석 도구를 이용해 추가 의사 결정 정보 획득
3-6. 최종 액션 의사 결정 전달

# 4. 보드 분석 도구(boardAnalysisTool.js)
5-1. 내 핸드와 보드가 얼마나 밀접하고 발전 가능성이 있는지 분석
5-2. 또는 얼마나 나에게 유리한 보드인지 분석
5-3. 점수 산정후 포스트 플랍 전략 레이어로 리턴

# 5. 빌런 프로파일링 도구(villainTool.js)
5-1. 상대 유효 스택
5-2. 상대 팟참여 비율 및 af 빈도 측정
5-3. 상대의 핸드레인지 유추
5-4. 점수 산정후 플랍 전략 레이어로 리턴
5-5. 상대방의 액션 히스토리(프리플랍 오픈레이즈, 나의 3BET, 상대방의 콜, 포스트 플랍 -> 나의 C-BET -> 상대의 레이즈 등)

# 6. 배팅 버짓 관리 도구(budgetTool.js)
6-1 자신의 스택과 빌런의 스택을을 기반으로 ?(또는 더 적절한 방안) 예산 편성
6-2. 만약 유리하다면 얼마나 배팅할것인지 어느 스트리트에 배분할 것인지 결정

# 7. 채팅(ChatTool.js 기존의 AIChatSystem.js를 그대로 써도 무방)
7-1 결정된 액션에 대한 대사 출력

데이터 흐름도
Hub가 게임 상태(State)를 읽어 context 객체 생성.
Tools들이 context를 받아 각각 점수(Score)와 분석 데이터 산출.
Strategies가 도구들의 분석 결과 + 핸드 레인지를 조합해 최적의 전략(Call, Raise 등) 도출.
Hub가 전략을 받아 최종적으로 게임 서버/클라이언트에 액션 명령 전달.
/aiEngine
 ├── aiBrainHub.js              // [Hub] 데이터 수집 및 최종 액션 결정
 ├── /strategies
 │    ├── aiBrainPreflop.js     // [Strategy] 프리플랍 GTO + 빌런 대응
 │    └── aiBrainPostflop.js    // [Strategy] 포스트플랍 에퀴티 기반 전략
 └── /tools
      ├── aiBrainBoardTool.js   // [Tool] 보드 텍스처 및 내 핸드 파워 분석
      ├── aiBrainVillainTool.js // [Tool] 상대 스탯 및 히스토리 프로파일링
      └── aiBrainBudgetTool.js  // [Tool] 팟 컨트롤 및 배팅 사이즈 계산
