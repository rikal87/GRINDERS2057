aiAgentTaskData.js의 데이터가 이전보다 훨씬 다양해짐에 따라,

SafeHouse.vue의 UI를 개선할 필요가 있습니다.
1. AgentTaskPopup.vue로 별도로 구현하여 App.vue에 컴포넌트형태로 연결
2. SafeHouse.vue에서 App.vue로 새로 만든 [AgentTaskPopup.vue]를 호출 하는 형태 (적절한 방법 $emit 또는 custom event 등으로 최적의 방안 고려 필요)
3. AgentTaskPopup.vue의 UI는 이제 아래와 같이 구성됩니다.


AI_TASK_DATA안에 데이터 구조는 아래와 같습니다.

{
    id: 'shadow_work_1',
    tier: 1,
    type: 'AGENT_WORK',
    name_ko: "서브넷 스캔 (합법)",
    name_en: "Sub-net Scan (Legal)",
    get name() { return store.settings.language === 'en' ? this.name_en : this.name_ko; },
    desc_ko: "기본적인 서브넷 스캔 작업입니다. 큰 수익은 없지만 안전합니다.",
    desc_en: "Basic sub-net scan task. No big profit, but safe.",
    get desc() { return store.settings.language === 'en' ? this.desc_en : this.desc_ko; },
    desc_detail_ko: "데이터베이스를 스캔하여 정보를 수집합니다.",
    desc_detail_en: 'Scans the database to collect information.',
    get desc_detail() { return store.settings.language === 'en' ? this.desc_detail_en : this.desc_detail_ko; },
    cost: 0,
    probability: 1,
    duration: 0,
    cooldown: 1 * 24 * 60,
    effect: [{ type: 'add_bankroll', amount: 1000 }],
    unlock: { type: 'none' }
  },

  여기서 `type` 변수에 따라 [AGENT_WORK, SEARCH, BOOST, NETWORKING] 탭 메뉴로 분리하여, 구성해야 합니다.
  - 각 탭안에 `tier` 변수에 따라 분류 된 TASK는 티어(tier)와 이름(name) 기준 오름차순으로 정렬.
  - 그외 세부 TASK UI 구성은 기존에 만들어진 SafeHouse.vue의 팝업 TASK SELECTOR MODAL [line 296~352] UI를 참고하여 구성하면 됩니다.
