현재 너무 WTSD에 매몰되어 수치 맞추기에만 혈안이 되어있다, 핵심 WTSD 수치의 활용방안을 다시 생각해야한다.
### 1. 콜링스테이션
1.id: 'mr_call name: 'MR_CALL', philosophy: 'Station', vPIP: .7, AF: 1, WTSD: .7
설계상 넓은 레인지로 콜만 하는 캐릭터이다. 그리고 약한 페어라도 맞추면 왠만해선 폴드가 불가하게 기획되어있음
perceivedEq = estimatedEquity + (WTSD - 0.33) * 1.2;
perceivedEq = estimatedEquity + (.7 - .33) * 1.2;
perceivedEq = estimatedEquity + .37 * 1.2;
perceivedEq = estimatedEquity + .444

2. WEAK PAIR estimatedEquity = .3
finalEq = EQ + perceivedEq = .3 + .444
finalEq = .744
2. MARGINAL EQ = .5 (보통 탑페어류)
finalEq = .5 + .444
finalEq = .944 의도대로 동작.

만약 어떤 NPC가 레이즈했고, mr_call이 콜을 해서 페어를 맞춘다면 쇼다운 까지 왠만해선 폴드하지 않는다 = 기획의도 (정상)

### 2. 타이트 어그레시브
id: 'Florence', name: 'Florence', fullName: 'Florence Quinn', philosophy: 'TAG', vPIP: .23, AF: 3.5, WTSD: .27

2. WEAK PAIR estimatedEquity = .3
perceivedEq = estimatedEquity + (WTSD - 0.33) * 1.2;
perceivedEq = estimatedEquity + (.27 - .33) * 1.2;
perceivedEq = estimatedEquity + -.06 * 1.2;
perceivedEq = estimatedEquity - .072

2. MARGINAL EQ = .5 (보통 탑페어류)
finalEq = .5 - .072
finalEq = .428

여기서 현재 문제점이 발견, 플랍에서 탑페어를 맞춰서 배팅을 할땐 상관없지만, 내가 수비입장이면

탑페어를 맞췃더라도 상대가 배팅할시 폴드해버리는 문제가 발생한다. 심지어 아주 작은 배팅이라도 말이다.

이렇기 때문에 저 에퀴티가 작동하는 시점을 새롭게 정의해야한다

가령 탑페어를 맞췄고 finalEq가 0.428이 되었다면, 추가적인 보호 에퀴티를 더해주는 방식이다.
이 에퀴티는 핸드 강함 평가에 영향을 주진않지만 일종의 생명선 역할을 하며 상대가 리레이즈를 하더라도 바로
일정 확률로 폴드하는 것이 아닌 콜로 받아칠 수 있게끔 말이다.
예시로 
finalEq가 0.428를 얻었다면
protectionEq 을 추가로 저장하고 있어서 상대가 내 배팅에 리레이즈를 하거나, 내가 콜을 해야하는 입장에서 보호 에퀴티를 차감하는 방식으로 동작하게 해야한다.
만약 이 보호 에퀴티가 모두 소진되면 일정확률로 폴드하는 방식으로 말이다.


// 예시 로직
let finalEq = estimatedEquity + (WTSD - 0.33) * 1.2;
// [사용자 제안: 보호 에퀴티 적용]
if (estimatedEquity >= 0.45) { // 탑 페어 이상급 강도라면
    // 아무리 니트(플로렌스)라도 탑 페어 이상이면 최소 85% 확률로는 일단 콜함
    const protectionFloor = 0.85; 
    finalEq = Math.max(finalEq, protectionFloor);
} 이렇게 짜면 상대의 트리플 배럴에서는 어떤 식으로 동작하지?
