# 개발 진척 사항 보고서 (2026.03.14)

## 📌 주요 업데이트 요약

지난 며칠 동안 **전체 UI 해상도 대응 시스템 고도화**, **세이프하우스(SafeHouse) UX 혁신**, **AI 에이전트 시스템 안정화** 및 **게임 코어 로직 오차 수정**을 진행했습니다. 특히 해상도와 관계없이 동일한 시각적 경험을 제공하는 '정비례 스케일링' 도입으로 시각적 완성도를 극대화했습니다.

1. **전체 해상도 정비례 스케일링 도입**: FHD(1080p)와 QHD(1440p) 간의 레이아웃 불일치를 해결하기 위해 뷰포트 높이 기반의 통합 스케일링(`1.25vh`) 시스템을 구축했습니다.
2. **세이프하우스 스테미나/수면 UI 통합**: 스테미나 바와 수면 조절 슬라이더를 하나로 합쳐, 수면 시 회복량을 실시간으로 예측할 수 있는 직관적인 인터페이스를 구현했습니다.
3. **AI 에이전트 및 작업 시스템 고도화**: 작업 취소 시 부스트 초기화 로직을 정교화하고, 쿨다운 상태에서의 조작 제한 및 텍스트 레이아웃을 최적화했습니다.
4. **코어 로직 및 밸런스 수정**: 카지노의 감시 상태(Monitoring)에 따른 의심도(Suspicion) 상승 로직의 반전 오류를 바로잡고, 9인 테이블 좌석 배치를 개선했습니다.

---

### 1. 🖥️ 해상도 독립적 정비례 스케일링 (`global.css`)
다양한 모니터 환경에서도 개발자의 의도대로 완벽한 16:9 비율을 유지할 수 있도록 설계를 개편했습니다.
* **통합 스케일링 기준 수립**: `1rem`을 뷰포트 높이의 일정 비율(`1.25vh`)로 고정하여, 창 크기에 따라 모든 UI 요소(폰트, 여백, 컴포넌트 크기)가 정비례하게 리사이징되도록 했습니다.
* **16:9 비율 보호망**: 울트라 와이드 모니터나 고해상도 환경에서도 전체 레이아웃이 옆으로 퍼지지 않도록 최대 너비를 높이의 1.77배(16:9)로 제한하는 보호 로직을 적용했습니다.
* **FHD/QHD 완벽 대응**: FHD 사용자가 QHD 사용자와 동일한 밀도의 UI 구성을 경험할 수 있도록 모든 `px` 기반 수치들을 `rem` 단위로 전환했습니다.

### 2. 🏠 세이프하우스(SafeHouse) 통합 UX 디자인
번거로운 조작을 줄이고 정보의 시각적 연결성을 강화했습니다.
* **스테미나-수면 슬라이더 통합**: 
  * 스테미나 바 자체를 슬라이더로 만들어 드래그만으로 수면 시간을 조절할 수 있습니다.
  * 조절 시 **'회복 예측 레이어(Preview)'**가 나타나 현재 스테미나 뒤로 얼마나 더 채워질지 실시간으로 보여줍니다.
* **콘텐츠 최적화**: 
  * AI 에이전트의 슬로건 가독성을 위해 줄바꿈 로직을 적용하고, QHD 전용 정보 노출 옵션을 강화했습니다.
  * 파트너 자산 변동 그래프(Sparkline)의 가시성을 위해 선 굵기를 정밀하게 조정했습니다.

### 3. ⚙️ AI 에이전트 작업(Task) 및 로직 안정화
* **작업 강제 취소 안정화**: 사용자가 활성 작업을 강제로 중단할 경우, 해당 작업과 연동된 모든 '액티브 부스트'가 즉시 초기화되도록 수정하여 데이터 무결성을 확보했습니다.
* **쿨다운(Cooldown) 제어**: 쿨다운 중인 작업은 UI상에서 물리적으로 조작을 차단하여 불필요한 오류를 방지했습니다.
* **감시 시스템 시스템 로직 수정 (`gameEngine.js`)**: 
  * "감시 중이 아닐 때" 의심도가 오르던 반전 버그를 수정하여, 카지노의 `isMonitoring` 상태가 `true`일 때만 의심도가 정상적으로 누적되도록 교정했습니다.

### 4. 🗺️ 로비 및 게임 테이블 레이아웃 보강
* **로비(Lobby) 미니맵 최적화**: 미니맵 뷰포트의 유동적 높이 조절 로직을 개선하여 다양한 해상도에서 더 넓은 시야를 확보했습니다.
* **9인 테이블 좌석 재배치 (`PokerTable.css`)**: 9인 풀 링 테이블의 좌석 배치를 타원형 전문 포커 레이아웃으로 재설계하여 플레이어 간 겹침 현상을 해결하고 가독성을 높였습니다.
