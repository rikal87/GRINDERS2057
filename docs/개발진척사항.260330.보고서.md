# [보고서] 개발 진척 사항 (2026.03.30)

## 1. AI 파트너 경제 시스템 재밸런싱 (Partner Economic Balancing)
- **생활비 시뮬레이션 개선**: 파트너의 무한 자산 증식을 방지하기 위해 비활성(도박 중이 아님) 시 시간당 자산의 **0.2% (최소 500 CR)**가 소모되도록 조정되었습니다.
- **레이크(Rake) 도입**: 도박 승리 시 배당금을 100%가 아닌 **95%**만 지급하여 실제 플레이어와 동일한 수수료 환경을 시뮬레이션합니다.
- **로직 최적화**: 도박 중에는 생활비가 중복 차감되지 않도록 조건문을 강화하여 비정상적인 손실을 방지했습니다.

## 2. 데이터 영속성 및 저장 시스템 최적화 (Persistence & Save Sync)
- **세션 진입 전 자동 저장**: 포커 테이블 입장 시(`handleJoinTable`) 현재 상태를 즉시 저장합니다. 이를 통해 플레이 중 앱 종료 시 세션 시작 전 상태로 안전하게 **롤백**할 수 있는 안전장치를 마련했습니다.
- **플레이 성능 개선**: 핸드가 끝날 때마다 발생하던 불필요한 `saveStore` 호출을 제거하여 게임 플레이 중 발생할 수 있는 데이터 병목 및 프레임 드랍을 방지했습니다.

## 3. UI/UX 및 모바일 반응형 레이아웃 강화
### AiAgentPopup (에이전트 관리 모달)
- **레이아웃 겹침 해결**: `modal-content-scroll` 레이어를 도입하여 내용이 길어져도 하단 "CLOSE/UPGRADE" 버튼 영역을 침범하지 않도록 수정했습니다.
- **HTML 구조 정규화**: 닫는 태그(`</div>`) 밸런스를 맞추고 구조를 재배치하여 모바일에서 레이아웃이 우측으로 밀리거나 겹치는 버그를 해결했습니다.
- **모바일 최적화**: `@media` 쿼리를 정돈하고 모바일 최대 높이를 **80vh**로 제한하여 작은 화면에서도 가독성을 확보했습니다.

### AgentTaskPopup (작업 배정 모달)
- **내부 스크롤 도입**: 모델 정보 및 상세 스태틱 영역에 개별 스크롤을 적용하여 전체 모달 구조가 깨지지 않도록 개선했습니다.
- **Flex 레이아웃 최적화**: 모바일 해상도에서 버튼 크기 및 간격을 세밀하게 조정하여 터치 UX를 개선했습니다.

## 4. 디자인 및 기타 세부 개선사항
- **SafeHouse 비주얼**: 패널 레이블 상단 여백(`margin-top`) 추가 및 QHD 고해상도(2560px)에서의 불필요한 외곽선 제거로 더 깔끔한 디자인을 적용했습니다.
- **효과음 교체**: 에이전트 장착 시 사운드를 일반 클릭음(`ui-click`)에서 `bootup` 사운드로 교체하여 사이버펑크 감성을 강화했습니다.
