# 개발 진척 사항 보고서 (2026.03.04)

## 📌 주요 업데이트 요약

오늘은 **초기 로딩 시 발생하는 순환 참조 에러 완벽 해결**, **세션 정산 리포트(PlayStatsPopup)의 UI/애니메이션 고도화 및 치명적 버그 수정**, 그리고 **글로벌 데이터 초기화 기능(Hard Reset) 신규 구현**을 중점적으로 진행했습니다.

1. **치명적인 순환 참조(Circular Dependency) 버그 해결**: 앱 로드 시 `store` 객체를 찾지 못해 터지던 화이트스크린 이슈를 종속성 분리를 통해 영구적으로 개선했습니다.
2. **세션 결산 애니메이션 & UI 강화**: 플레이어의 눈길을 끄는 순차적인 팝업 및 숫자 카운팅 트랜지션 애니메이션을 도입하고, 내부적인 텍스트 포맷 버그를 모조리 잡아냈습니다.
3. **사용자 편의성(QoL) 및 AI Sizing 로직 보강**: 로비 맵 뷰포트 개선, 홀드(Hold) 방식의 세이브 초기화 버튼 신설, 그리고 AI의 프리플랍 스퀴즈(Squeeze) 벳 사이즈 로직을 다듬었습니다.

---

### 1. ⚙️ 코어 아키텍처 안정화 (순환 참조 고리 절단)
앱의 최초 로드 시점을 불안정하게 만들던 Vue Store의 의존성 지옥을 완전히 해소했습니다.
* **`store.js` <-> `persona.js` 간의 상태 의존성 분리:**
  * 페르소나 데이터 파싱 중 다국어 지원을 위해 `store.settings`를 다이렉트로 요구하면서 발생하던 `Cannot access 'store' before initialization` 에러를 수정했습니다.
  * `persona.js` 내부에 빈 껍데기 콜백(Closure)인 `setLanguageGetter`를 만들어두고, `store.js`가 무사히 준비된 이후에 언어 설정 함수를 역으로 주입(Inject)해주는 고급 패턴을 적용해 순환 참조 구조를 깔끔하게 끊어냈습니다.
* **파트너(Partner) 초기화 패턴 현대화:**
  * 무거웠던 `Partner` 객체 클래스를 날려버리고 가벼운 팩토리(Factory) 객체 패턴으로 리팩토링하여 하이드레이션(Hydration) 낭비를 줄였습니다.
  * `store`의 State에 파트너 객체가 하드코딩 되어 할당되지 않고 초기화 시동(`initStore`) 과정에서 안전하게 주입되도록 생명주기를 늦췄습니다.

### 2. 📊 정산 리포트 (PlayStatsPopup) 시각화 및 안정화
세션 종료 후 플레이어가 마주하는 정산 피드백 인터페이스의 심미성과 데이터의 안정성을 크게 끌어올렸습니다.
* **통계 카운팅(Number Roll) 및 순차 애니메이션 도입:**
  * 한 번에 딱딱하게 나타나던 통계들이 `ECONOMIC_METRICS` ➡️ `BEHAVIORAL_HUD` ➡️ `LUCK_AND_PEAKS` 순서로 부드럽게 나타나도록 페이즈 트랜지션을 주었습니다.
  * 숫자가 0에서 목표치까지 서서히 상승하는 보간 애니메이션(`animateValue`) 로직을 추가하여 리포트를 보는 타격감을 더했습니다. 더불어 `BigInt` 자료형이 애니메이션 로직에 포함되며 터지던 `TypeError`를 철저히 캐스팅(`Math.floor`, `Number()` 변환)하여 예방했습니다.
* **정산 메시지(MSG_CODE) 맵핑 버그 및 예외 처리:**
  * `msgCode`의 대소문자가 불일치(`WIN_BIG` vs `win_big`)하여 피드백 메시지가 증발(출력되지 않음)하던 이슈의 원인을 찾아내 일괄 교체했습니다.
  * 특정 상태(`NEUTRAL` 등)에서 랜덤 메시지 텍스트 배열을 찾지 못하고 `undefined` 값을 꺼내려다 Vue 템플릿 단계에서 앱이 뻗어버리던 치명적 약점을 방어하는 예외 처리(`!msgObj` 시 `''` 반환 로직) 시스템을 견고하게 추가했습니다.

### 3. 편의성 증진 및 AI / UI 소규모 픽스
* **데이터 하드 리셋(HARD RESET) 버튼 신규 구현:**
  * 세팅창에 사용자의 진행 상황(저장된 데이터베이스)을 초기화하는 `RESET_GAME_DATA` 버튼을 도입했습니다. 
  * 실수클릭을 방지하기 위해 3초 동안 마우스/터치를 유지(`Hold`)해야만 프로그레스 바가 차오르고 초기화가 확정 실행되도록 안전 장치 로직을 심어두었습니다.
* **AI 엔진 코어 – 프리플랍 림프/스퀴즈 규격 보정 (`aiEngine.js`):**
  * 앞선 사람들이 콜만 한 상황(Limp Pot)에서 뒤에 있는 "고급(Advanced)" 등급 AI가 레이즈를 칠 때, 단순히 블라인드의 일정 배수만 치는게 아니라 팟에 이미 뿌려진 콜러들의 칩(Pot Size)까지 알뜰하게 더합하여 올바른 스퀴즈(Squeeze) 및 오버벳 사이즈가 나오도록 연산식을 고도화했습니다.
* **로비(Lobby) UI 및 전역 최적화:**
  * 로비 전역의 중복되는 칩 잔액 표시(`CREDITS_AVAILABLE`) 레이아웃을 다듬고 텍스트 규격을 최적화했습니다.
  * 미니맵 텍스처를 새로운 V4 모델 스킨으로 교체하고 스캐너 오퍼시티 타겟 및 패딩 뷰포트를 손봤습니다.
